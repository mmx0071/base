# PR 预览平台 — 流程图

> 镜像构建统一使用 **GitHub Actions**。Minikube 与正式环境控制语义一致，差异在 Webhook 入口与 Registry。

---

## 1. 总览架构

```mermaid
flowchart LR
    subgraph GitHub
        PR[pull_request]
        GHA[GitHub Actions\npreview-build]
    end
    subgraph 集群外
        GHA --> REG[(Registry)]
        GHA -->|image-ready| WH
    end
    subgraph 控制面
        PR -->|Webhook| WH[preview-webhook]
        WH --> CR[PreviewEnvironment CR]
        CR --> OP[preview-operator]
    end
    subgraph 数据面_preview_NS
        OP --> DEP[Deploy / Svc / Ingress / NP]
        REG -->|pull| DEP
        DEP --> URL[预览 URL]
    end
```

---

## 2. 镜像构建（GitHub Actions）

```mermaid
flowchart TB
    START{pull_request\ntype?}
    START -->|opened / reopened / synchronize| RUN[启动 preview-build.yml]
    START -->|closed| END1[不构建]

    RUN --> C1[checkout PR head SHA]
    C1 --> C2[计算 image tag\npr-n-shortSHA]
    C2 --> C3[docker build + push Registry]
    C3 -->|失败| F1[workflow 失败\n不回调]
    C3 -->|成功| C4[POST image-ready]

    C4 --> R{HTTP 状态}
    R -->|200/204| OK[结束]
    R -->|404| RETRY[退避 2/4/8/16/32s 重试]
    RETRY --> C4
    R -->|401/403/409/5xx| F2[workflow 失败]

    WH2[synchronize 时 Webhook\n清空 spec.image] -.->|并行| RUN
```

## 3. PR 生命周期（双通路时序）

```mermaid
sequenceDiagram
    participant GH as GitHub
    participant WH as preview-webhook
    participant CR as CR
    participant CI as GitHub Actions
    participant REG as Registry
    participant OP as preview-operator

    GH->>WH: opened → Create CR image=""
    GH->>CI: 触发 build
    par
        WH->>CR: spec 写入
    and
        CI->>REG: push 镜像
    end
    CI->>WH: image-ready
    WH->>CR: patch spec.image
    OP->>CR: Reconcile → Ready
    Note over GH,OP: synchronize：WH 清空 image → Pending → 再 build → image-ready
    GH->>WH: closed → Delete CR
    OP->>CR: Finalizer 清理 preview NS 资源
```

## 4. Minikube 流程

```mermaid
flowchart TB
    P0[阶段0\nminikube + Ingress + tunnel + hosts]
    P1[阶段1\n开发 Operator / Webhook\nimage load 进 minikube]
    P2[阶段2\n部署 CRD / RBAC / Operator / Webhook\n创建 Secrets]
    P3[阶段3\n演示仓 + GHCR\nport-forward + ngrok\nGitHub Webhook + Actions Secrets]
    A[阶段A\n开 PR → Actions → image-ready → curl 预览]
    B[阶段B\nRBAC / NP / 关 PR 清理]

    P0 --> P1 --> P2 --> P3 --> A --> B

    subgraph 运行时_每次PR
        R1[GitHub PR] --> R2[ngrok → Webhook]
        R2 --> R3[CR Pending]
        R3 --> R4[Actions → GHCR]
        R4 --> R5[image-ready → ngrok]
        R5 --> R6[Operator → Ingress\npreview.local]
    end

    P3 -.-> 运行时_每次PR
```

## 5. 正式环境流程

```mermaid
flowchart TB
    subgraph 一次性_平台
        D1[安装 CRD / RBAC / NP]
        D2[部署 Operator + Webhook]
        D3[Ingress + TLS + 环境变量\nREGISTRY / PREVIEW_DOMAIN]
    end

    subgraph 每仓库接入
        R1[preview-build.yml]
        R2[Registry 凭据 + Callback Secrets]
        R3[GitHub Webhook → 固定域名]
        R4[allowedRepos + imagePullSecret]
    end

    subgraph 每次_PR
        T1[opened → Webhook → CR Pending]
        T2[Actions → 企业 Registry]
        T3[image-ready → patch image]
        T4[Operator → Ready\npr-n-repo.preview.domain]
        T5[closed → 删 CR → Finalizer 清理]
    end

    D1 --> D2 --> D3 --> R1
    R1 --> R2 --> R3 --> R4 --> T1
    T1 --> T2 --> T3 --> T4
    T4 --> T5
```

## 6. Operator Reconcile（两环境相同）

```mermaid
flowchart TD
    IN[Watch PreviewEnvironment] --> DEL{deletionTimestamp?}
    DEL -->|是| FIN[Finalizer 有序删子资源\nIngress→Deploy→Svc→NP→CM→Secret]
    DEL -->|否| IMG{spec.image 空?}
    IMG -->|是| PEND[phase = Pending\n不建 Deployment]
    IMG -->|否| CREATE[创建 NP / CM / Secret / Deploy / Svc / Ingress]
    CREATE --> READY{就绪?}
    READY -->|是| R[phase = Ready]
    READY -->|超时| F[phase = Failed]
```
